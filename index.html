myProjects

"T.O.P. Resume App"


Here I am, back into react and vite after a very long time. Despite a major false start when I tried to incorporate future lessons into this project (LOL, don't do that!), this project got way more fun once I started really learning the best practices of using react. My best advice here, take the time to really think about how you want to divide the components up. (Mobile ready.)

"T.O.P. Homepage"


This was the most fun I've had on one of these projects in some time. The hardest part was the design, as usual. Like always, I overly enjoyed creating silly SVG assets, and trying to make my homepage different from the more standard stuff everyone else is coding. There is nothing better in life than wasting a few hours tinkering with art, it's probably why I like coding too. This was the very first project in which I added aria-labels (now that I know what they are). (Mobile ready.)

"T.O.P. Battleship Game"


This was a beast, and the last of the many Javascript section projects. Working on this entailed the same sad mix of stubborn perfectionism, curiosity, and a desire to get almost all the ideas that were in my head into working code. Aside from refusing to do the bare minimum, the real headache on this project was ensuring that all features worked on the Brave mobile and Apple Safari browsers. Either way, the game is pretty entertaining, and I'm a bit too good at Battleship strategy now. Enjoy. (Mobile ready.)

"T.O.P. Weather App"


Another project that ate up more time than I should have permitted. Getting the app to function was not the hard part, rather, designing it to meet my own picky expectations was, and it caused a lot of changing design headaches. That said, this will definitely be an app that I use daily. This was so much fun to code up and is my favorite T.O.P. app to date. (Mobile ready.)

"T.O.P. To-Do List"


This project took a long time to finish. Once again the design phase lures me in and I dream up functionality that I'd ideally want in a good to-do list app. The problem is then trying to implement this "wish list" into code. Like all of these projects, I bit off too much and learned some great, but hard lessons. Looking at you window.location.reload, which I need to stop using. (Mobile ready.)

"T.O.P. Restaurant Page"


This project introduces the fun world of webpack to the coding mix. The app itself is for a fictional bookstore in fictional Arkham, MA. Multiple pages are available on the app, each handling informational aspects that might be required of a real bricks and mortar establishment. (Mobile ready.)

"T.O.P. Tic-Tac-Zodiac"


A simple tic-tac-toe game that has a single-player mode against the computer, or a two-player mode for you and a friend. I had wanted to include a more "competent" computer opponent, but the minimax algorithm code was just not working out within my time constraints. This will be a fun project to come back too. (Mobile ready.)
